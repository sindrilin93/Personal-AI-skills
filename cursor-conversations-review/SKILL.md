# 对话回顾总结 Skill

## When to use

在以下场景触发该 Skill：

- 用户要求“对话复盘 / 回顾总结 / session 分析”
- 用户表达类似意图：
  - “帮我复盘一下最近的对话”
  - “总结一下我这段时间怎么和你交流的”

---

该 Skill 用于对最近一段时间的 Cursor 对话进行结构化复盘，包括：

- 对话提取
- Session 分类
- 提问质量分析
- 回答有效性分析
- 优化建议输出

## 执行步骤

### Step 1：提取最近对话

执行以下脚本：

```bash
python3 ./cursor_conversation_extract.py
```

要求：

- 脚本会自动输出最新对话内容到：

```text
./reviews/yyyy-MM-dd.md
```

- 读取当天最新生成的文件。
- 如果文件不存在，提示用户“脚本执行失败，未生成复盘输入文件”，并停止后续步骤。

### Step 2：读取对话内容并进行 Session 分类

读取上述 markdown 文件，对对话进行分析。

分类规则：

- 将对话按“回答内容关联性”划分为多个 session。
- 区分一次性问答和连续问答。

**剔除近期对话总结的问答**

后续分析将基于对话 session 分析，session 分类使用表格输出：

- session 问题
- 目标
- 关键结果
- 参考会话

### Step 3：分析提问质量（用户侧）

识别以下问题类型：

1. 不恰当提问（目标不清）
2. 冗余提问（重复 / 低价值）
3. 缺乏约束（没有边界条件）
4. 信息不足（缺少上下文）

输出格式：

```markdown
## ❗ 提问优化建议

| 提问 | 存在问题 & 原因 | 优化建议 & 示例 |
| --- | --- | --- |
| <原提问> | <问题类型 + 原因> | <改写建议 + 示例> |
```

### Step 4：分析 Cursor 回答质量（模型侧）

识别以下问题类型：

1. 无效回答（未解决问题）
2. 偏离需求
3. 过度泛化
4. 未被用户采纳（用户明确否定或忽略）

分析维度（必须给出归因）：

- 提问问题（用户侧）
- 模型理解问题
- 缺少约束条件
- Prompt 不清晰

输出格式：

```markdown
## 🤖 回答质量分析

| 问题 | 回答摘要 | 问题分类 & 原因 | 优化建议 |
| --- | --- | --- | --- |
| <用户问题> | <回答摘要> | <分类 + 归因> | <改进建议> |
```

## 执行约束

- 必须严格按照步骤执行，不可跳步。
- 所有分析必须基于提取的对话内容，不允许凭空假设。
- 输出必须结构化（Markdown）。
- 不要省略任何模块。

## 最终输出结构

最终输出必须严格包含：

```markdown
# 🧠 对话回顾总结（yyyy-MM-dd）

## 📦 Session 分类
...

## ❗ 提问优化建议
...

## 🤖 回答质量分析
...
```

## 目标

该 Skill 的目标不是总结内容，而是：

- 提升提问质量
- 提高 AI 输出有效性
- 建立长期可复用的 Prompt / Rule 体系
